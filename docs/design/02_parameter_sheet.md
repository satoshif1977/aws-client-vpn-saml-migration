## ① VPN エンドポイント基本設定

| パラメータ | 設定値 | 状態 | 備考 |
|---|---|---|---|
| エンドポイントID | `cvpn-endpoint-XXXXXXXXXXXXXXXXX` | ✅ | 既存エンドポイント（PoC用） |
| エンドポイント名 | `client-vpn-poc-dev` | ⚠️ | 命名規則を確認 |
| クライアント CIDR ブロック | ⚠️ 確認要 | ⚠️ | VPC・社内NWと重複しない /22 以上 |
| トランスポートプロトコル | UDP（国内）/ TCP（海外拠点） | ⚠️ | 海外拠点は GFW 対策で TCP 443 推奨 |
| VPN ポート | 443 | ⚠️ | 海外拠点はポート制限の可能性あり |
| スプリットトンネル | 有効（推奨） | ⚠️ | 有効にすると社内宛のみ VPN 経由 |
| DNS サーバー | ⚠️ 確認要 | ⚠️ | 社内 DNS サーバーの IP を設定 |
| セルフサービスポータル | 有効 | ⚠️ | ユーザーが自分で .ovpn を取得可能 |
| セッションタイムアウト | 24時間 | ⚠️ | セキュリティポリシーに合わせて調整 |

## ② 認証設定（SAML / OneLogin 連携）

| パラメータ | 設定値 | 状態 | 備考 |
|---|---|---|---|
| 認証タイプ | フェデレーション認証（SAML 2.0） | ✅ | OneLogin を IdP として使用 |
| IdP（ID プロバイダー） | OneLogin | ✅ | 既存 OneLogin 環境を流用 |
| SAML プロバイダー ARN | 🔧 作業時に取得 | 🔧 | IAM Identity Provider として登録後に確定 |
| ACS URL | `http://127.0.0.1:35001` | ✅ | AWS VPN Client がローカルで受け取る URL |
| Audience URI（SP Entity ID） | `urn:amazon:webservices:clientvpn` | ✅ | AWS 固定値 |
| MFA | 有効（OneLogin 側で設定済み） | ✅ | OneLogin の MFA ポリシーに依存 |
| SAMLアサーション属性名 | `memberOf` | ✅ | AWS Client VPN の認可で使用 |

### OneLogin 側 SAML アプリ設定

| パラメータ | 設定値 | 状態 |
|---|---|---|
| アプリ名 | AWS Client VPN | ⚠️ 既存アプリの有無を確認 |
| SAML署名アルゴリズム | SHA-256 | ✅ |
| SAML Initiator | Service Provider | ✅ |
| ロール属性マッピング | ロール名 → `memberOf` | ⚠️ 設定済みか確認 |

## ③ ネットワーク設定

| パラメータ | 設定値 | 状態 | 備考 |
|---|---|---|---|
| VPC ID | ⚠️ 確認要 | ⚠️ | Client VPN を接続する VPC |

| セキュリティグループ | ⚠️ 確認要 | ⚠️ | Client VPN 用 SG を新規作成 |

### セキュリティグループ設計（案）

| 方向 | プロトコル | ポート | 送信元/宛先 | 用途 |
|---|---|---|---|---|
| インバウンド | UDP / TCP | 443 | 0.0.0.0/0 | VPN クライアントからの接続 |
| アウトバウンド | ALL | ALL | 社内ネットワーク CIDR | 社内リソースへのアクセス |
