# STEP 5：海外拠点接続テスト手順書（PoC）

---

## テスト概要

| 項目 | 内容 |
|---|---|
| 目的 | AWS Client VPN で海外拠点から社内システムへ接続できることを確認 |
| 使用する .ovpn | `client-config-overseas.ovpn`（TCP 443 版） |
| 前提 | STEP 4（国内テスト）合格済みであること |

---

## 海外拠点テスト特有の注意点

```
① GFW（グレートファイアウォール）
   → UDP がブロックされる可能性 → TCP 443 を優先で試す

② 時差（例：中国 UTC+8 / 日本 UTC+9）
   → テスト実施時間の調整が必要

③ 切り戻しリスク
   → テスト中も旧 VPN を残しておく（問題発生時に即切り戻し）
```

---

## 事前確認チェックリスト

- [ ] テストユーザーが OneLogin アカウント + MFA 設定済み
- [ ] テストユーザーが対応する OneLogin ロールに割り当て済み
- [ ] AWS Client VPN エンドポイントが起動中
- [ ] `client-config-overseas.ovpn`（TCP 443 版）を配布済み
- [ ] AWS VPN Client インストール済み
- [ ] **旧 VPN は切断せず残しておく**（切り戻し用）

---

## STEP 5-1：VPN 接続（TCP 443）

1. AWS VPN Client を起動 → `client-config-overseas` を選択 → `Connect`
2. ブラウザが開き OneLogin ログイン画面が表示される
3. ID・パスワード入力 → MFA 認証完了
4. `Connected` と表示されることを確認

**【日本側確認】**
- CloudWatch Logs に接続ログが出力されていることを確認

---

## STEP 5-2：疎通確認

```
# AD サーバー（全グループ共通）
ping 10.2.1.X    # AD-GUEST-01
ping 10.2.1.X    # AD-GUEST-02

# グループ別アクセス確認（⚠️ グループ確定後に記入）
ping [アクセス先IP]
```

| 確認項目 | 期待結果 | 合否 |
|---|---|---|
| AD-GUEST-01 への ping | 応答あり | ⬜ |
| AD-GUEST-02 への ping | 応答あり | ⬜ |
| グループ許可範囲内への ping | 応答あり | ⬜ |
| グループ許可範囲外への ping | タイムアウト | ⬜ |

---

## STEP 5-3：業務システムへのアクセス確認

| 確認対象 | 期待結果 | 合否 |
|---|---|---|
| ⚠️ 業務システム（グループ確定後に記入） | ログイン画面表示 | ⬜ |
| 許可範囲外システム | アクセス拒否 | ⬜ |

---

## STEP 5-4：切断・再接続確認

1. `Disconnect` → `Disconnected` を確認
2. 再度 `Connect` → OneLogin 認証 → `Connected` を確認

---

## 問題発生時の対応フロー

```
問題発生 → 日本側担当者に連絡
  ↓
CloudWatch Logs でエラー確認
  ↓
  30分以内に解決できそう？
    Yes → 原因調査・修正して再テスト
    No  → 旧 VPN に切り戻し
```

### よくあるエラーと対処

| エラー症状 | 原因 | 対処法 |
|---|---|---|
| ブラウザが開かない | UDP がブロックされている | TCP 443 の .ovpn で試す |
| Connected 後アクセス拒否 | memberOf 属性未設定 / グループ名不一致 | CloudWatch Logs で確認 |
| 一部 IP にのみ接続できる | 認可ルールの CIDR 設定漏れ | 対象 CIDR の認可ルールを追加 |

---

## 切り戻し手順

1. AWS VPN Client → `Disconnect`
2. 旧 VPN クライアントで接続
3. 業務システムにアクセスできることを確認
4. 日本側担当者に切り戻し完了を連絡
